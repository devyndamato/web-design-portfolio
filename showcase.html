Project Overview

I built this portfolio website to present my experience, skills, and selected work in public relations, strategic communications, and related creative projects. The goal was to create a clean, organized site that helps visitors quickly understand who I am, what kind of work I have done, and how to contact me. I designed the site with recruiters, hiring managers, professors, and potential collaborators in mind, focusing on clear navigation, readable content, and a polished visual style.

Project Details

  • Project: Personal Portfolio Website
  • Purpose: Showcase my work, experience, and professional interests in public relations and strategic communications
  • Audience: Recruiters, hiring managers, professors, and potential collaborators
  • Role: Designer, writer, and developer
  • Tools Used: HTML, CSS, JavaScript, VS Code, GitHub Pages, and Figma

Why I Built It

I created this website as a central place for my academic, internship, and professional work. Instead of keeping my experience spread across separate documents or platforms, I wanted one site that clearly communicates my background, strengths, and interests. Each page was designed to be easy to scan while still showing professionalism, personality, and strong organization.

Tools and Skills Used

  • HTML for content structure and page organization
  • CSS for typography, color, spacing, layout, and responsive design
  • JavaScript for interactive elements like the menu toggle and reveal features
  • VS Code for writing and editing code
  • GitHub and GitHub Pages for version control and publishing the live site
  • Figma for planning layout and wireframe ideas

How I Built It

I built this project in stages, starting with content planning and wireframes before moving into coded page layouts. As the project developed, I focused on improving visual consistency, strengthening the design system, and making the site more responsive across screen sizes. I also revised older pages so the site felt more professional and less like an early class project. Throughout the process, I practiced organizing content for real audiences, writing cleaner HTML, improving accessibility basics, and using CSS and JavaScript more intentionally.

What I Learned

This project helped me better understand how content, design, and code work together to shape a user’s experience. I learned how important it is to build with both structure and audience in mind, especially when creating a site meant to represent me professionally. I also became more comfortable with responsive design, debugging, and refining details like navigation, spacing, accessibility, and overall consistency.
